  <files>src/layouts/Layout.astro, src/pages/index.astro</files>
  <action>
    Both files currently exist as empty stubs (1 line each). Write their content.

    **src/layouts/Layout.astro** — Import `framer/styles.css` in the frontmatter (between `---` fences), NOT in a `<style>` tag. Astro scopes `<style>` blocks, which would prevent Framer's CSS variables from cascading to child components. The frontmatter import is treated as a global stylesheet.

    ```astro
    ---
    import '../components/framer/styles.css';
    ---
    <!DOCTYPE html>
    <html lang="en">
      <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Colton Carroll</title>
      </head>
      <body>
        <slot />
      </body>
    </html>
    ```

    No SEO meta tags, no OG tags — that is Phase 4 scope.

    **src/pages/index.astro** — A minimal stub that uses Layout so the build has a valid home page. Phase 2 will replace the content.

    ```astro
    ---
    import Layout from '../layouts/Layout.astro';
    ---
    <Layout>
      <main>
        <p>Portfolio coming soon.</p>
      </main>
    </Layout>
    ```

    Also remove `src/components/FramerComponent.tsx` and `src/components/Header.astro` if they are still empty stubs — these are placeholders that will cause confusion in Phase 2 when real Framer components exist.
