  <files>tailwind.config.js</files>
  <action>
    Create `tailwind.config.js` in the project root using CommonJS `module.exports` syntax (NOT ESM `export default`). `@astrojs/tailwind` v6 loads the config via `require()` — ESM syntax will break the build.

    Exact content:
    ```js
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      content: ['./src/**/*.{astro,html,js,jsx,md,mdx,ts,tsx}'],
      darkMode: 'class',
      theme: {
        extend: {},
      },
      plugins: [],
    };
    ```

    `darkMode: 'class'` is required so Framer's CSS variable dark mode tokens respond to a `dark` class on `<html>`. The `content` glob covers all Astro, React, and Markdown source files.
  </action>
  <verify>
    <automated>node -e "const c=require('./tailwind.config.js'); if(c.darkMode!=='class') { console.error('darkMode not class'); process.exit(1); } console.log('OK');"</automated>
